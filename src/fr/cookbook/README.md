Bienvenue sur la documentation officielle de Stone.js où les jalons d'une grande aventure viennent à peine d'être posés. 

Stone.js est un microframework web vous permettant de decouvrir ou redecouvrir la beauté, la simplicité et la puissance de vanilla JavaScript, 
en mettant à votre disposition des outils simple et efficaces afin de créer des applications web performante pouvant se deployer partout où l'on peut executer du JavaScript.

Stone.js est très modulaire, très leger, très portatif et independant des plate-formes(Navigateur, Node.js, Severless, Worker, etc.), il permet de créer:
- des microservices
- des applications Cloud natif sans serveur
- des API HTTP
- des applications WebSocket
- des applications CLI
- des applications web classiques(monolithique) rendues coté serveur
- des applications pouvant s'executer sur le navigateur
- des applications pouvant s'executer dans un web worker ou service worker.

**Note:** Pour l'instant il n'est pas encore possible de faire du developpement frontend(SPA, SSR, Jamstack, etc.) avec Stone.js.

## Par où commencer?

Il est important de bien comprendre la [philosophie du framework](./introduction) avant d'aller plus loin, ensuite libre à vous de choisir l'un des deux scenarios suivants.

Si vous êtes du genre à aimer se salir les mains, à plonger directement dans le code, il est mieux de commencer ici: [**Getting started**](./introduction/).

Mais si vous êtes du genre à aimer passer du temps à appréhender les concepts avant de se salir les mains, il est mieux de commencer ici: [**Concepts de base**](./architecture/).

Une fois que vous ayez compris la philosophie et les concepts de base du framework, 
libre à vous d'aller directement dans la section repondant à votre besoin.

## Structure de la documentation

- [**Getting started**](./introduction/) est un guide vous permettant de comprendre le framework, de l'installer, de le configurer et de l'utiliser
- [**Concepts de base**](./architecture/) vous permet de comprendre toute l'architecture de Stone.js, la raison d'etre de chacune des composantes et leur interdependance
- [**Essentials**](./essentials/) contient tous les outils transversaux et essentiels à presque tous les differents types d'application(Web, API, Serverless, CLI)
- [**HTTP**](./http/) contient tous les outils necessaire au developpement d'une application HTTP
- [**Digging Deeper**](./deeper/) contient des outils plus ou moins avancés et non obligatoire au developpement d'une application
- [**Database**](./database/) contient tous les outils necessaire pour interagir avec une base données
- [**Security**](./security/) contient tous les outils necessaire pour assurer la sécurité au sein d'une application
- [**Testing**](./testing/) contient tous les outils necessaire afin d'effecuter des tests.
