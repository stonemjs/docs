    What is Stone.js?

    Stone.js is a future-ready framework designed for full-context development:
    cloud, serverless, browser, and beyond.

    Continuum Architecture

    One mindset across backend, frontend, serverless, and browser. Learn once, build anything.

    Universal by Design

    Run on Node.js, AWS Lambda, Browser, or anywhere, with zero friction.

    Modular & Compact

    No node_modules. One artifact. Pure ESM. Built for performance and portability.

    Adaptive Blueprint API

    Configure apps declaratively or imperatively. No boilerplate. Full control.

    First-Class CLI

    Create, build, test and deploy with one simple and powerful command line.

    Cloud Native Ready

    Built with modern development in mind: SSR, FaaS, edge computing, HMR, lazy loading, and more.

    Multi-Dimensional Runtime

    Stone.js executes across multiple dimensions, functional, integration, and view, while preserving internal logic.

    Zero Context Leakage

    Strict separation of domain and context using adapters and blueprints. Your app stays clean, testable, and portable.

    Get Started with Stone.js

    From zero to cloud-native in under a minute. Stone.js was built for this.

    @NodeHttp()
@StoneApp()
export class Application implements IEventHandler<IncomingEvent> {
  private readonly logger: ILogger

  constructor ({ logger }: { logger: ILogger }) {
    this.logger = logger
  }

  handle(event: IncomingEvent): { message: string } {
    const message = `Hello ${event.get<string>('name', 'World')}!`

    this.logger.info(message)

    return { message }
  }
}

    @Router()
@NodeHttp()
@StoneApp()
@EventHandler()
export class Application {
  @Get({ path: '/' })
  welcome(event: IncomingEvent): { message: string } {
    const message = `Hello ${event.get<string>('name', 'World')}!`
    return { message }
  }

  @Get({ path: '/bye' })
  bye(event: IncomingEvent): { message: string } {
    const name = event.get<string>('name', 'World')
    const message = `Good Bye ${name}!`
    return { message }
  }
}

    Continuum in Action

    Stone.js isn't built on layers, it's built on a flow, from raw events to rendered experiences. That flow is called the Continuum.

    Integration

    Convert external requests into contextual events (from Lambda, Node, or browser).

    Initialization

    Register configs, services, plugins, prepare your app for logic.

    Functional

    Handle intentions using event handlers and return responses.

    View

    Render your response in React or Vue, same logic, different surface.
